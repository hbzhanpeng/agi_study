## 1. GPT 系列演进路线图

| 模型 | 发布时间 | 参数量 | 核心能力 | 训练数据 |
|------|---------|--------|---------|---------|
| GPT-1 | 2018.06 | 1.17 亿 | 预训练+微调 | 7000 本书（5GB） |

- GPT-3：证明"模型足够大，给几个示例就能做复杂任务"

## 2. GPT-1：预训练+微调范式

### 2.1 核心创新

**两阶段训练**：
```
阶段 1：预训练（Pre-training）
- 数据：海量无标注文本（如维基百科）
- 任务：预测下一个词（语言建模）
- 目标：学会语言的通用知识

阶段 2：微调（Fine-tuning）
- 数据：少量标注数据（如 1000 条情感分类）
- 任务：具体的下游任务
- 目标：适配特定应用
```

**前端类比**：
- 预训练 = 学习 JavaScript 基础（通用能力）
- 微调 = 学习 React 框架（具体应用）

### 2.2 预训练目标：语言建模

**任务**：给定前面的词，预测下一个词

```
输入："我 喜欢 吃"
模型预测："苹果"（0.3）、"饭"（0.25）、"肉"（0.2）...

为什么有效？
- 要预测下一个词，必须理解：
  - 语法（"吃"后面接名词）
  - 语义（"喜欢吃"后面是食物）
  - 常识（苹果是常见食物）
```

### 2.3 架构：Transformer Decoder

**关键特点**：
- 只用 Decoder 部分（12 层）
- 单向注意力（只能看左边，看不到右边）
- 为什么？预测下一个词时不能"偷看"答案

```
句子："我 喜欢 吃 苹果"

预测"吃"时：
✅ 能看到："我 喜欢"
❌ 不能看到："苹果"（这是要预测的）
```

### 2.4 面试考点

**问法 1**："GPT 的预训练目标是什么？"

**标准答案**：
> "GPT 使用语言建模作为预训练目标，即给定前面的词序列，预测下一个词。通过这个自监督任务，模型能从海量无标注文本中学习语言的语法、语义和常识知识。"

**问法 2**："预训练和微调有什么区别？"

**标准答案**：
> "预训练阶段用海量无标注数据学习通用语言表示，微调阶段用少量标注数据适配具体任务。预训练提供了强大的初始化，使得微调只需要很少的数据就能达到好效果。"

## 3. GPT-2：Zero-Shot Learning

### 3.1 核心突破

**Zero-Shot Learning**：不给任何示例，直接让模型做任务

```
任务：情感分类

传统方法（需要微调）：
- 先给 1000 条标注："这电影真好看 → 正面"
- 微调模型
- 然后才能分类

GPT-2 的 Zero-Shot：
输入："这电影真好看。情感："
输出："正面"

没有微调！直接做！
```

### 3.2 任务即提示（Task as Prompt）

**关键洞察**：把任务描述直接写在输入里

```
翻译任务：
输入："英文：I love you. 中文："
输出："我爱你"

问答任务：
输入："问题：中国的首都是哪里？答案："
输出："北京"

摘要任务：
输入："文章：[长文本] TL;DR:"
输出："[摘要]"
```

**前端类比**：
```javascript
// 传统方法：为每个任务写专门函数
function translateToEnglish(text) { ... }
function translateToChinese(text) { ... }

// GPT-2 风格：通用函数 + Prompt 控制
function universalModel(prompt) {
  // 根据 prompt 格式自动判断任务
}
```

### 3.3 涌现能力（Emergent Abilities）

**发现**：模型变大后，出现了训练时没有明确教过的能力

```
GPT-2 在预训练时：
- 没有专门学过"翻译"
- 没有专门学过"摘要"
- 没有专门学过"问答"

但因为见过大量网页文本（包含各种格式），
自然学会了这些能力！
```

### 3.4 为什么能做到？

1. **数据规模**：40GB 网页文本（是 GPT-1 的 10 倍）
2. **模型规模**：15 亿参数（是 GPT-1 的 10 倍）
3. **隐式学习**：预训练数据中包含各种任务格式

### 3.5 面试考点

**问法**："Zero-Shot Learning 是什么？"

**标准答案**：
> "Zero-Shot Learning 是指模型在没有见过任何任务示例的情况下，直接执行该任务。GPT-2 通过在海量文本上预训练，学会了理解任务描述（Prompt），从而能够在不微调的情况下完成各种任务。"

## 4. GPT-3：Few-Shot Learning

### 4.1 核心飞跃

**Few-Shot Learning**：给模型几个示例，不微调也能达到接近微调的效果

**对比三种学习方式**：

| 方式 | 示例数量 | 是否微调 | 效果 | 成本 |
|------|---------|---------|------|------|
| Fine-tuning | 1000+ | ✅ 需要 | 最好（~90%） | 高 |
| Few-Shot | 5-10 | ❌ 不需要 | 接近微调（~85%） | 低 |
| Zero-Shot | 0 | ❌ 不需要 | 较差（~70%） | 最低 |

**例子：情感分类**

```
Few-Shot Prompt：
"这部电影很精彩。情感：正面
这部电影很无聊。情感：负面
这部电影一般般。情感：中性
这电影真好看。情感："

输出："正面"
准确率：~85%（接近微调的 90%）
```

### 4.2 In-Context Learning（上下文学习）

**定义**：模型通过输入中的示例，动态理解任务，无需参数更新

**类比**：
```
传统机器学习 = 考试前复习（微调参数）
In-Context Learning = 开卷考试（看着例题做题）
```

**为什么 GPT-3 能做到？**
- 预训练时见过海量的"模式识别"场景
- 模型学会了"从示例中推断规律"的元能力
- 这是**涌现能力**的极致体现

**关键**：学习发生在**推理时**，不更新参数

### 4.3 规模法则（Scaling Law）

**核心发现**：模型越大，Few-Shot 效果越好

```
GPT-1（1.17 亿参数）：Few-Shot 几乎不行
GPT-2（15 亿参数）：Few-Shot 勉强能用
GPT-3（1750 亿参数）：Few-Shot 接近微调

结论：性能 ∝ 模型规模（参数、数据、算力）
```

**临界点**：
- 100B 参数以下：Few-Shot 效果一般
- 100B 参数以上：Few-Shot 效果显著提升
- 这是**涌现能力**的体现

### 4.4 Few-Shot Prompt 设计技巧

**基础格式**：
```
示例 1：输入 → 输出
示例 2：输入 → 输出
示例 3：输入 → 输出
待预测：输入 → 
```

**优化技巧**：

1. **示例数量**：2-5 个最佳（太少效果差，太多超长度限制）
2. **格式统一**：所有示例格式必须一致
3. **覆盖边界**：包含正常情况 + 边界情况
4. **任务描述**：开头加任务说明，提升准确率
5. **中英混合**：任务描述用英文，内容用中文（效果更好）

**示例：人名提取**

```
任务：从句子中提取所有人名。

句子：小明去游泳了。
人名：小明

句子：小刚和小亮坐在椅子上玩游戏。
人名：小刚、小亮

句子：今天天气真好。
人名：无

句子：李明和王芳一起去了北京。
人名：
```

### 4.5 面试考点

**问法 1**："Few-Shot Learning 和 Fine-tuning 有什么区别？"

**标准答案**：
> "Few-Shot Learning 是在推理时给模型几个示例，让它通过上下文学习完成任务，不需要更新参数。Fine-tuning 是用标注数据训练模型，需要更新参数。Few-Shot 的优势是快速、灵活，不需要大量标注数据；劣势是效果通常不如 Fine-tuning。"

**问法 2**："什么是 In-Context Learning？"

**标准答案**：
> "In-Context Learning 是指大语言模型通过输入中的示例，动态理解并执行任务，无需参数更新。这是 GPT-3 等超大模型的涌现能力，模型在预训练时学会了'从示例中推断规律'的元能力。"

**问法 3**："为什么 GPT-3 能做 Few-Shot，GPT-2 不行？"

**标准答案**：
> "主要是规模差异。GPT-3 有 1750 亿参数，是 GPT-2 的 100 倍。更大的模型有更强的泛化能力和上下文学习能力，能够从少量示例中快速推断任务模式。这是涌现能力的体现，当模型规模达到临界点后，性能会有质的飞跃。"

## 6. 关键理解

### 6.1 预训练的本质

**不是**：记住所有知识
**而是**：学会语言的通用模式和推理能力

```
类比：
预训练 ≠ 背字典
预训练 = 学会如何理解和使用语言
```

### 6.2 涌现能力的本质

**不是**：人为设计的功能
**而是**：规模达到临界点后自然出现的能力

```
小模型：1 + 1 = 2（机械记忆）
大模型：理解"加法"的概念，能做任意加法（泛化能力）
```

### 6.3 Prompt Engineering 的本质

**不是**：调参技巧
**而是**：如何用自然语言与模型沟通

```
传统编程：用代码告诉计算机做什么
Prompt Engineering：用自然语言告诉模型做什么
```

## 7. 真实项目经验

### 经验 1：Zero-Shot vs Few-Shot 的选择

**场景**：客服意图识别

```
Zero-Shot：
- 准确率：~65%
- 成本：几乎为 0
- 适用：快速验证、低要求场景

Few-Shot（5 个示例）：
- 准确率：~82%
- 成本：人工标注 5 条
- 适用：中等要求场景

Fine-tuning（1000 条）：
- 准确率：~91%
- 成本：人工标注 1000 条 + GPU 训练
- 适用：高要求、大规模场景
```

**结论**：根据准确率要求和成本预算选择

### 经验 2：Prompt 设计的坑

**坑 1：格式不一致**
```
❌ 错误示例：
"这电影好看。情感：正面"
"这电影难看 → 负面"
"这电影一般 = 中性"

✅ 正确示例：
"这电影好看。情感：正面"
"这电影难看。情感：负面"
"这电影一般。情感：中性"
```

**坑 2：示例质量差**
```
❌ 错误：都是简单情况
"好。情感：正面"
"差。情感：负面"

✅ 正确：覆盖复杂情况
"虽然剧情一般，但演员演技不错。情感：中性"
"前半段很好，后半段崩了。情感：负面"
```

**坑 3：中文 Prompt 效果差**
```
❌ 纯中文：准确率 75%
"任务：提取人名"

✅ 中英混合：准确率 82%
"Task: Extract person names from Chinese sentences."
```

### 经验 3：线上事故

**事故**：客服机器人回复不当

**原因**：Zero-Shot Prompt 没有覆盖边界情况
```
用户："你们公司是骗子！"
机器人（Zero-Shot）："感谢您的反馈！"
```

**修复**：改用 Few-Shot，加入负面情绪示例
```
用户："你们公司是骗子！"
机器人（Few-Shot）："非常抱歉给您带来不好的体验，我立即为您转接人工客服。"
```

**教训**：生产环境必须用 Few-Shot + 充分测试
