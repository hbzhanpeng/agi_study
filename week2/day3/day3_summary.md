## 1. 核心问题：计算机怎么理解文字？

**挑战**：计算机只认识数字，怎么表示"猫"、"狗"这些词？

### 方案对比

| 方案 | 表示方式 | 优点 | 缺点 |
|-----|---------|------|------|
| 编号 | 猫=1, 狗=2, 汽车=3 | 简单 | 无法表达相似度 |
| 词向量 | 猫=[0.8,0.9,0.1], 狗=[0.7,0.85,0.15] | 能表达语义相似度 | 需要训练 |

**结论**：词向量能让相似的词在数字空间中也接近

---

## 2. 词向量（Word Embedding）

### 2.1 核心思想

**用一组数字（向量）表示一个词，让语义相似的词向量接近**

```
猫    = [0.8, 0.9, 0.1, 0.2]
狗    = [0.7, 0.85, 0.15, 0.25]  ← 和猫接近
汽车  = [0.1, 0.2, 0.9, 0.8]     ← 和猫狗远离
```

### 2.2 神奇特性：向量运算表达语义关系

```
king - man + woman ≈ queen
Paris - France + Italy ≈ Rome
```

**原因**：词向量捕捉了词之间的语义关系

---

## 3. Word2Vec（经典词向量算法）

### 3.1 核心思想

**一个词的意思，由它周围的词决定**

```
"猫在沙发上睡觉"
"狗在沙发上睡觉"
→ "猫"和"狗"上下文相似 → 词向量接近

"汽车在马路上行驶"
→ "汽车"上下文不同 → 词向量远离"猫狗"
```

### 3.2 局限性

**问题**：一个词只有一个固定向量，无法区分一词多义

```
"我喜欢吃苹果"（水果）
"我喜欢用苹果手机"（品牌）

Word2Vec：两个"苹果"是同一个向量 ❌
```

---

## 4. 注意力机制（Attention）⭐⭐⭐

### 4.1 核心思想

**让模型知道"重点看哪里"，根据上下文动态调整词的表示**

### 4.2 人类的注意力

```
句子："猫在沙发上睡觉"

理解"睡觉"时：
- 重点关注"猫"（谁在睡？）← 注意力权重高
- 关注"沙发"（在哪睡？）← 注意力权重中
- 不太关注"在"、"上"  ← 注意力权重低
```

### 4.3 注意力机制的计算

**目标**：理解"苹果"在句子中的真正含义

**句子 1**："我喜欢吃苹果"
```
苹果 vs 吃：关系很大 → 注意力权重 0.7

苹果的新向量 = 
  0.1 × "我"的向量 +
  0.2 × "喜欢"的向量 +
  0.7 × "吃"的向量

→ 苹果的向量偏向"食物"语义
```

**句子 2**："我喜欢用苹果手机"
```
苹果 vs 手机：关系很大 → 注意力权重 0.6

苹果的新向量 = 
  0.1 × "我"的向量 +
  0.2 × "喜欢"的向量 +
  0.1 × "用"的向量 +
  0.6 × "手机"的向量

→ 苹果的向量偏向"品牌"语义
```

### 4.4 解决的问题

1. **一词多义**：根据上下文动态调整词的表示
2. **长距离依赖**：可以关注句子中任意位置的词

---

## 5. Word2Vec vs BERT

| 特性 | Word2Vec | BERT |
|-----|----------|------|
| 词向量类型 | 静态（固定） | 动态（根据上下文） |
| 一词多义 | ❌ 无法区分 | ✅ 可以区分 |
| 核心技术 | 上下文窗口 | 注意力机制 |
| 训练方式 | 预测周围词 | 双向编码 |

## 8. 关键理解

**词向量的本质**：
- 把词映射到高维空间
- 相似的词在空间中距离近
- 能用数学运算表达语义关系

**注意力机制的本质**：
- 计算词与词之间的关系强度
- 根据关系强度动态调整词的表示
- 让模型"看懂"上下文

**为什么重要**：
- 词向量是所有 NLP 模型的基础
- 注意力机制是 Transformer 的核心
- 不理解这两个，后面的 GPT、BERT 都听不懂
