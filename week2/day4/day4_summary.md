## 2. Self-Attention 完整计算流程

### 2.1 Q、K、V 的含义

**每个词都有三个向量**：

| 向量 | 含义 | 类比 |
|-----|------|------|
| Query（查询）| 我要找什么信息？ | 图书馆搜索词 |
| Key（键）| 我能提供什么信息？ | 书的标签 |
| Value（值）| 我的实际内容是什么？ | 书的内容 |

**计算流程**：
1. 用 Query 和所有 Key 计算相关性（点积）
2. 找到最匹配的 Key
3. 读取对应的 Value

### 2.2 完整计算步骤

**句子**："我 喜欢 吃 苹果"

**步骤 1：计算注意力分数**
```
苹果的 Query · 我的 Key = 0.1
苹果的 Query · 喜欢的 Key = 0.3
苹果的 Query · 吃的 Key = 0.9  ← 最高
苹果的 Query · 苹果的 Key = 0.5
```

**步骤 2：Softmax 归一化**
```
[0.1, 0.3, 0.9, 0.5] 
→ Softmax →
[0.05, 0.15, 0.6, 0.2]  ← 注意力权重
```

**步骤 3：加权求和**
```
苹果的新表示 = 
  0.05 × "我"的 Value +
  0.15 × "喜欢"的 Value +
  0.6  × "吃"的 Value +      ← 权重最高
  0.2  × "苹果"的 Value

结果：苹果的向量融合了上下文，偏向"食物"语义
```

### 2.3 完整公式

```
Attention(Q, K, V) = Softmax(Q·K^T / √d_k) · V

其中：
- Q·K^T：计算相关性分数（点积）
- √d_k：缩放因子
- Softmax：转换成概率分布
- ·V：加权求和
```

## 3. 为什么要除以 √d_k？

### 3.1 问题：点积太大

```
当维度 d_k = 512 时：
Q·K = 0.5×0.4 + 0.3×0.2 + ... (512 项相加)
    ≈ 100 多（很大的数字）

Softmax([100, 95, 90]) ≈ [0.99, 0.01, 0.00]
                          ↑
                    注意力过于集中！
```

### 3.2 后果

- 注意力几乎全在一个词上，其他词信息丢失
- Softmax 饱和，梯度消失，训练困难

### 3.3 解决方案

```
缩放后：Q·K / √512 ≈ 100 / 22.6 ≈ 4.4

Softmax([4.4, 4.2, 4.0]) ≈ [0.4, 0.35, 0.25]
                            ↑
                      注意力分布更均匀
```

**数学原理**：点积的方差 ∝ d_k，除以 √d_k 让方差稳定在 1

## 4. Multi-Head Attention（多头注意力）

### 4.1 单头的局限

```
单头注意力：只能从一个角度看问题
- 苹果主要关注"吃"（语义关系）
- 但可能忽略"喜欢"（情感关系）
```

### 4.2 Multi-Head 的优势

```
让模型同时从多个角度看问题：

Head 1：关注语义关系（苹果 ← 吃）
Head 2：关注情感关系（苹果 ← 喜欢）
Head 3：关注位置关系（苹果在句末）
...
Head 8：关注其他模式

最后拼接所有头的输出 → 更全面的理解
```

### 4.3 类比：多专家会诊

```
单头 = 一个医生看病（只能从一个角度）
多头 = 8 个专家会诊（综合多个角度）
```

### 4.4 结构

```
输入 → 分成 8 个头

Head 1: Q1,K1,V1 → Attention1 → 输出1
Head 2: Q2,K2,V2 → Attention2 → 输出2
...
Head 8: Q8,K8,V8 → Attention8 → 输出8

拼接 → 线性变换 → 最终输出
```

## 5. 位置编码（Positional Encoding）

### 5.1 问题：Self-Attention 没有位置信息

```
句子 A："狗 咬 人"
句子 B："人 咬 狗"

Self-Attention 计算的注意力权重一样！
因为它只看词与词的关系，不管顺序
```

**更多例子**：
- "我爱你" vs "你爱我" → 被认为一样
- "大狗" vs "狗大" → 被认为一样

### 5.2 解决方案：位置编码

```
在输入时加上位置信息：

狗 = [0.5, 0.8, 0.3] + [位置0的编码]
咬 = [0.6, 0.7, 0.4] + [位置1的编码]
人 = [0.4, 0.9, 0.2] + [位置2的编码]

现在模型能区分：
"狗(位置0) 咬(位置1) 人(位置2)" ≠ "人(位置0) 咬(位置1) 狗(位置2)"
```

### 5.3 位置编码的实现

Transformer 使用正弦/余弦函数：
```
PE(pos, 2i) = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))
```

**优点**：
- 可以处理任意长度的序列
- 相对位置关系可以通过三角函数性质表达

## 6. Transformer 完整流程

```
输入："我 喜欢 吃 苹果"

1️⃣ 词嵌入（Word Embedding）
   每个词 → 向量

2️⃣ 位置编码（Positional Encoding）
   向量 + 位置信息

3️⃣ Multi-Head Self-Attention
   - 8 个注意力头并行计算
   - 每个头从不同角度看上下文
   - 拼接所有头的输出

4️⃣ 残差连接 + 层归一化
   Add & Norm

5️⃣ 前馈网络（Feed Forward）
   两层全连接网络

6️⃣ 残差连接 + 层归一化
   Add & Norm

7️⃣ 重复 N 次（如 12 层）
   每层提取更高级的特征

输出：每个词的上下文表示
```

## 7. 面试考点

### 问题 1：Q、K、V 的作用，为什么除以 √d_k？

**标准答案**：
> "Q（Query）是查询向量，代表当前词想要找什么信息；K（Key）是键向量，代表每个词能提供什么信息；V（Value）是值向量，代表词的实际内容。
> 
> 通过 Q 和 K 的点积计算相关性，然后用注意力权重对 V 加权求和。
> 
> 除以 √d_k 是为了缩放点积值。当维度很大时，点积会很大，导致 Softmax 饱和，注意力过于集中，同时会导致梯度消失。"

### 问题 2：Multi-Head 的优势

**标准答案**：
> "Multi-Head Attention 让模型能从多个子空间同时捕捉不同类型的依赖关系。不同的头可以关注不同的模式：语义、语法、位置等。最后拼接所有头的输出，得到更全面的表示。"

### 问题 3：为什么需要位置编码

**标准答案**：
> "Self-Attention 本身是位置无关的，只计算词与词的关系，不考虑顺序。这会导致'我爱你'和'你爱我'被认为一样。位置编码在输入时给每个词加上位置信息，让模型能理解词序。"

## 8. 关键理解

**Self-Attention 的本质**：
- 让每个词"看到"其他所有词
- 根据相关性动态调整自己的表示
- 解决了 RNN 的长距离依赖问题

**Multi-Head 的本质**：
- 多个专家从不同角度分析
- 每个头学习不同的注意力模式
- 综合多个角度得到更丰富的表示

**位置编码的本质**：
- 补充 Self-Attention 缺失的位置信息
- 让模型能理解词序对语义的影响
