# 第 4 周 Day 1 学习总结：RAG 核心架构

> **核心目标：掌握 RAG 的三大阶段和核心价值。**

---

## 1. 什么是 RAG

**Retrieval-Augmented Generation（检索增强生成）**

一句话：**先搜相关文档，再让 LLM 基于文档回答**。

### RAG 解决 LLM 的两个核心问题

| 问题 | 说明 | RAG 怎么解决 |
|------|------|-------------|
| 知识截止 | 模型不了解训练后的新数据和私有数据 | 从知识库检索最新信息 |
| 幻觉 | 模型可能"一本正经胡说" | 回答基于真实文档，可溯源验证 |

---

## 2. RAG 三大阶段（面试必画）

```
阶段 1: Indexing（离线索引）
  文档 → 分块(Chunking) → Embedding → 存入向量数据库

阶段 2: Retrieval（在线检索）
  用户提问 → Embedding → 向量相似度搜索 → 返回 Top K 文档块

阶段 3: Generation（生成回答）
  [相关文档块] + [用户问题] → LLM → 生成最终回答
```

---

## 3. RAG vs Fine-tuning

| 维度 | RAG | Fine-tuning |
|------|-----|-------------|
| 数据更新 | ✅ 实时（重新索引） | ❌ 需重新训练 |
| 成本 | ✅ 低 | ❌ 高（GPU） |
| 可解释性 | ✅ 可溯源 | ❌ 黑盒 |
| 幻觉控制 | ✅ 基于文档 | ❌ 仍可能幻觉 |

---

## 4. 面试标准话术

> 「RAG 是检索增强生成，先从知识库检索相关文档，再交给 LLM 生成回答。它解决了两个问题：一是知识截止，二是幻觉。好处是不需要微调，新数据只要重新索引即可。」
