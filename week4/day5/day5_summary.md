# 第 4 周 Day 5 学习总结：RAG 进阶技巧

> **核心目标：掌握 RAG 效果优化的三大技巧。**

---

## 1. Query 改写（Query Rewriting）

**问题**：用户问题太口语化，检索不到  
**方案**：用 LLM 改写成专业查询再检索

```
"年假咋整？" → LLM 改写 → "公司年假政策是什么？" → 检索 ✅
```

---

## 2. HyDE（Hypothetical Document Embeddings）

**问题**：问题和文档语义空间不对齐（问题 vs 答案）  
**方案**：先让 LLM 生成假回答，用假回答去检索

```
问题："年假多少天？"
  ↓ LLM 生成假回答
假回答："员工入职满一年后享有15天年假"
  ↓ 用假回答去检索（假回答语义格式=真文档格式）
匹配更精准 ✅
```

---

## 3. Self-RAG（自适应检索）

**问题**：不是所有问题都需要检索  
**方案**：模型自己判断3件事

1. 需要检索吗？（"1+1=?" → 不需要）
2. 检索结果有用吗？
3. 回答有幻觉吗？

---

## 4. RAG 优化层次（面试标准答法）

```
层次 1: Chunking 优化（调分块大小和 Overlap）
层次 2: Query 改写（口语→专业）
层次 3: HyDE（假回答检索）
层次 4: Hybrid Search + Rerank
层次 5: Self-RAG（自适应）
```

**关键**：对症下药，不盲目叠技巧

---

## 5. 实战：Query 改写

在 `rag_demo.py` 中实现了 `rewrite_query()` 函数：

```python
# 改写 prompt ≠ 回答 prompt！
# 改写只处理问题本身，不需要 context
prompt = f"请将以下口语化问题改写为专业搜索查询。\n原始问题：{question}\n改写后："
```

### 踩坑记录
- ❌ 把 RAG 回答 prompt 复制过来用（带 context）→ 报错，且职责不对
- ✅ 改写 prompt 只需要 question，不需要 context

### 关键理解
- **改写 prompt** = "翻译官"（口语→书面语）→ 不需要文档
- **RAG prompt** = "回答官"（基于文档回答）→ 需要文档
