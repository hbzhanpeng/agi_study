# 第 4 周 Day 3 学习总结：Embedding 模型选型 + 向量数据库

> **核心目标：掌握 Embedding 和向量数据库的选型策略。**

---

## 1. Embedding 模型选型

| 模型 | 中文效果 | 成本 | 适用场景 |
|------|---------|------|---------|
| OpenAI text-embedding-3 | 好 | 付费 | 快速上线 |
| **BGE-large-zh** | ✅ 最强 | 免费开源 | **中文场景首选** |
| M3E | 很好 | 免费开源 | 轻量中文 |
| bge-m3 | 多语言强 | 免费开源 | 中英混合 |

### 选型决策
```
中文为主 → BGE-large-zh
多语言   → bge-m3
快速上线 → OpenAI
```

---

## 2. 向量数据库选型

| 数据库 | 类比 | 适用场景 |
|--------|------|---------|
| **Chroma** | SQLite | 本地开发、快速原型 |
| **Milvus** | MySQL | 生产环境、大规模 |
| Pinecone | AWS RDS | 托管免运维（⚠️ 数据出境）|
| FAISS | 内存数组 | 离线实验 |

### ⚠️ 关键经验
- **国内敏感行业（法律/金融/政务）= 排除海外 SaaS**
- Pinecone 数据存海外，不满足数据不出境要求
- 必须选 Milvus 等可私有化部署的方案

---

## 3. 索引算法

| 算法 | 特点 |
|------|------|
| **HNSW** | 速度快、精度高、最常用 ✅ |
| IVF | 适合超大数据集 |
| PQ | 省内存但精度降 |

## 4. 面试考点

- **Embedding 选型**：根据语言+场景，中文首选 BGE
- **向量数据库选型**：开发 Chroma，生产 Milvus
- **加分项**：在自己数据集上 benchmark，不看排行榜选
- **合规意识**：敏感行业必须私有化部署
