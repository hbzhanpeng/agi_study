## 🎯 今日核心要点

### 为什么对齐如此重要？

**ChatGPT 的成功 = 预训练 + RLHF**

预训练模型的问题：
- 只学会了"预测下一个 token"
- 不知道什么是"好答案"
- 会输出有害内容、谎言、答非所问

对齐的价值：
- 让模型输出符合人类价值观
- 提高指令遵循能力
- 减少幻觉和有害输出

---

## 📖 知识点 1：为什么需要对齐

### 3H 原则（面试必背）

| 原则 | 含义 | 例子 |
|------|------|------|
| **Helpful** | 有帮助 | 真正解决用户问题 |
| **Harmless** | 无害 | 拒绝有害请求 |
| **Honest** | 诚实 | 承认不确定，不编造 |

### 预训练 vs 对齐

| 维度 | 预训练 | 对齐 |
|------|--------|------|
| 目标 | 学习语言模式 | 学习人类偏好 |
| 数据 | 海量文本（TB） | 人类标注（万-十万） |
| 学到什么 | "下一个词是什么" | "什么是好答案" |
| 成本 | 极高（千万美元） | 中等（十万美元） |

## 📖 知识点 2：RLHF 原理

### RLHF 三阶段

```
阶段 1：SFT（监督微调）
    ↓ 1-10 万条问答数据
    ↓ 学会基本指令遵循
    
阶段 2：RM（奖励模型）
    ↓ 10-100 万条偏好数据
    ↓ 学会判断"什么是好答案"
    ↓ 输出：标量分数（不是文本！）
    
阶段 3：PPO（强化学习）
    ↓ 用 RM 指导训练
    ↓ 让模型输出高奖励的答案
```

### 奖励模型（RM）

**关键点**：
- 输入：prompt + response
- 输出：标量分数（如 0.85）
- **不生成文本！**

**训练方式**：
```python
loss = -log(sigmoid(reward(chosen) - reward(rejected)))

目标：让 reward(chosen) > reward(rejected)
```

### PPO 损失函数

```
loss = -reward + β × KL_divergence(new_policy, old_policy)

其中：
- reward：奖励模型给的分数
- KL_divergence：防止偏离参考模型太远
- β：KL 惩罚系数（0.01-0.1）
```

**为什么需要 KL 惩罚？**
- 防止模型为了高奖励输出乱码（reward hacking）
- 保持模型的语言能力

### 数据需求

| 阶段 | 数据类型 | 数据量 | 标注成本 |
|------|---------|--------|---------|
| SFT | 问答对 | 1-10 万 | 高（$5/条） |
| RM | 偏好对比 | 10-100 万 | 中（$0.5/条） |
| PPO | 无需新数据 | - | 低（自动） |

**关键洞察**：
- RM 数据便宜 10x（只需比较，不需要写答案）
- PPO 不需要新数据（模型自己生成）

## 📖 知识点 3：DPO 简化方案

### DPO 核心思想

> "奖励模型只是中间步骤，可以直接从偏好数据推导出最优策略"

**RLHF 的问题**：
```
偏好数据 → 训练 RM → 用 RM 指导 PPO → 更新模型
         ↑ 中间多了一步，增加复杂度
```

**DPO 的改进**：
```
偏好数据 → 直接优化模型
         ↑ 省掉了 RM
```

### DPO vs RLHF 对比

| 维度 | RLHF | DPO |
|------|------|-----|
| 阶段数 | 3 步（SFT → RM → PPO） | 2 步（SFT → DPO） |
| 模型数 | 4 个 | 2 个 |
| 显存需求 | 4 × 模型 | 2 × 模型 |
| 训练稳定性 | 不稳定 | 稳定 |
| 训练速度 | 慢（10x） | 快（接近 SFT） |
| 效果 | 最好（100%） | 接近（95%） |
| 实现难度 | 高 | 低 |

### DPO 损失函数

```python
loss = -log(σ(β × log(π(y_w|x)/π_ref(y_w|x)) 
              - β × log(π(y_l|x)/π_ref(y_l|x))))

直观理解：
- 增大 chosen 的概率（相对于参考模型）
- 减小 rejected 的概率（相对于参考模型）
```

### DPO 超参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| beta | 0.1-0.5 | 控制偏离参考模型的程度 |
| learning_rate | 5e-7 到 1e-6 | 比 SFT 小 100x |
| loss_type | sigmoid | 标准 DPO |

## 🔥 数据类型决定方法（关键洞察）

### 数据类型对比

| 数据类型 | 格式 | 用于 | 标注成本 |
|---------|------|------|---------|
| 问答对 | `{prompt, response}` | SFT | 高（$5/条） |
| 偏好对比 | `{prompt, chosen, rejected}` | RLHF/DPO | 低（$0.5/条） |

### 选型决策树

```
有偏好数据吗？
  ├─ 否 → 只能做 SFT
  ↓
  是
  ↓
资源充足吗？（4+ 张 A100）
  ├─ 否 → DPO（资源有限）
  ↓
  是
  ↓
追求极致效果吗？
  ├─ 是 → RLHF（效果最好）
  ├─ 否 → DPO（性价比高）
```

### 关键误区（今日错题）

❌ **错误理解**：有偏好数据 → 只做 SFT  
✅ **正确理解**：有偏好数据 → 应该用 RLHF/DPO

**原因**：
- 偏好数据是专门为对齐设计的
- 包含了"什么是好答案"的信息
- 只做 SFT 浪费了偏好数据的价值

## 💼 面试真题演练

### 题目 1：RLHF 三阶段

**面试官**："RLHF 的三个阶段分别做什么？"

**标准回答**：
> "RLHF 分三个阶段：
> 
> **阶段 1：SFT（监督微调）**
> - 用 1-10 万条高质量问答数据微调
> - 让模型学会基本的指令遵循能力
> 
> **阶段 2：RM（奖励模型）**
> - 用 10-100 万条偏好对比数据训练
> - 训练一个模型判断'什么是好答案'
> - 输出标量分数，不生成文本
> - 数据成本比 SFT 低 10x
> 
> **阶段 3：PPO（强化学习）**
> - 用奖励模型指导训练
> - 模型自己生成数据，无需新标注
> - 损失函数：`-reward + β × KL_divergence`
> - KL 惩罚防止 reward hacking"

### 题目 2：DPO vs RLHF

**面试官**："DPO 和 RLHF 的区别是什么？什么时候用哪个？"

**标准回答**：
> "**核心区别**：
> - RLHF：3 步（SFT → RM → PPO），需要 4 个模型
> - DPO：2 步（SFT → DPO），只需 2 个模型
> - DPO 直接从偏好数据优化，不需要奖励模型
> 
> **效果对比**：
> - RLHF：效果最好（100%）
> - DPO：接近 RLHF（95%）
> 
> **选型建议**：
> - 资源充足 + 追求极致 → RLHF
> - 资源有限 + 快速迭代 → DPO
> - 大部分场景 → DPO（性价比高）
> 
> **成本对比**：
> - RLHF：4 个模型，显存 4x，训练慢 10x
> - DPO：2 个模型，显存 2x，训练快（接近 SFT）"

### 题目 3：奖励模型

**面试官**："奖励模型是怎么工作的？它生成文本吗？"

**标准回答**：
> "奖励模型**不生成文本**，它只输出一个标量分数。
> 
> **工作流程**：
> 1. 输入：prompt + response
> 2. 通过 Transformer（从 SFT 模型初始化）
> 3. 通过线性层
> 4. 输出：标量分数（如 0.85）
> 
> **训练方式**：
> - 用偏好对比数据训练
> - 损失函数：`-log(sigmoid(reward(chosen) - reward(rejected)))`
> - 目标：让好答案的分数 > 差答案的分数
> 
> **架构对比**：
> - 生成模型：Transformer → LM Head → 词表概率
> - 奖励模型：Transformer → Linear → 标量分数"

### 题目 4：为什么需要对齐

**面试官**："为什么 ChatGPT 比 GPT-3 好用那么多？"

**标准回答**：
> "核心区别在于 **RLHF 对齐**。
> 
> **GPT-3 的问题**：
> - 只做了预训练，学会了预测 token
> - 不知道什么是'好答案'
> - 会输出有害内容、编造信息、答非所问
> 
> **ChatGPT 的改进**：
> - 加入了 RLHF 对齐
> - 遵循 3H 原则：Helpful、Harmless、Honest
> - 大幅提升指令遵循能力和安全性
> 
> **量化对比**：
> - 指令遵循能力提升 ~40%
> - 有害输出减少 ~80%
> - 用户满意度显著提升
> 
> 这就是为什么 ChatGPT 能成为现象级产品。"

## 🔥 工程经验总结

### RLHF 的挑战

1. **复杂度高**：需要维护 4 个模型
2. **训练不稳定**：PPO 容易崩溃
3. **超参数难调**：KL 系数、学习率等
4. **成本高**：显存 4x，训练时间 10x

### DPO 的优势

1. **简单**：只需 2 个模型
2. **稳定**：监督学习，不会崩溃
3. **快**：训练速度接近 SFT
4. **效果好**：接近 RLHF 的 95%

### 最佳实践

1. **数据质量最重要**：
   - SFT 数据要高质量（1-10 万条）
   - 偏好数据要多样（10-100 万条）
   - chosen 和 rejected 差异要明显

2. **选型建议**：
   - 90% 的场景用 DPO 就够了
   - 只有追求极致效果才用 RLHF

3. **超参数**：
   - DPO beta：0.1（标准值）
   - DPO lr：5e-7（比 SFT 小 100x）
   - RLHF KL 系数：0.05
