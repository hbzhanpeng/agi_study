# 第 1 周 Day 1 学习总结：Python 基础语法与数据结构

> **核心目标：跨越语法门槛，掌握 Python 四大核心数据结构，能用 Python 思考逻辑。**

## 1. 基础语法对比
* **前端类比**：
  * 没有 `{}` 和 `;`，靠**缩进**（通常是 4 个空格）划分代码块，非常考验代码格式。
  * `let/const` 声明变量变成了直接赋值 `name = "AGI"`。
  * `console.log()` 变成了 `print()`。
  * `True/False/None` 首字母必须大写（对应 JS 的 `true/false/null`）。

## 2. 四大核心数据结构
这是 Python 面试最喜欢问的基石，几乎所有 AI 数据的处理都在用这四个结构。

* **List (列表 `[]`)**：
  * **对应 JS**：`Array`
  * **特点**：有序，可变。
  * **常用操作**：`append()`, `pop()`, 切片 `lst[0:3]`（非常强大，类似 `slice` 但语法更简练）。

* **Dict (字典 `{}`)**：
  * **对应 JS**：`Object` / `Map`
  * **特点**：键值对，查找极快。
  * **常用操作**：`user["name"]` 或安全的 `user.get("name", "默认值")`，`user.keys()`。

* **Set (集合 `set()`)**：
  * **对应 JS**：`Set`
  * **特点**：无序，元素唯一，非常适合做**数据去重**或**求交集/并集**。
  * **面试考点**：如何快速去除列表中的重复项？答：`list(set(my_list))`。

* **Tuple (元组 `()`)**：
  * **对应 JS**：没有直接对应的结构，类似于**只读（Readonly）的 Array**。
  * **特点**：一旦创建就不能修改（不可变类型）。
  * **为什么需要它**：当你想把多个值打包当成字典的 key 时（List 不能当 key，但 Tuple 可以），或者保护数据不被意外修改时。

## 3. 推导式 (Comprehensions)
* **前端类比**：类似于将 JS 的 `map()` 和 `filter()` 合二为一，并且语法更扁平。
* **为什么重要**：这是 Python 特有的“优雅写法（Pythonic）”，写 AI 处理脚本时极为常用。
* **核心代码**：
  ```python
  # 需求：取列表里的偶数，并把它们平方
  nums = [1, 2, 3, 4, 5, 6]
  
  # JS 写法： nums.filter(n => n % 2 === 0).map(n => n * n)
  
  # Python 推导式： [生成值 for 变量 in 集合 if 条件]
  squared_evens = [n * n for n in nums if n % 2 == 0] 
  ```
