# 第 3 周 Day 5 学习总结：多模态 API + 模型选型策略

> **核心目标：掌握多模态 API 用法和模型选型的工程思维。**

---

## 1. 多模态 API

| 模态 | 能力 | 典型用法 |
|------|------|---------|
| Vision | 图片理解 | OCR、截图分析、拍照识物 |
| Audio | 语音↔文字 | 语音助手、会议记录 |
| Image Gen | 文生图 | DALL-E、Midjourney |
| Video | 视频理解 | 视频摘要、内容审核 |

### API 调用格式
```json
{
  "messages": [{
    "role": "user",
    "content": [
      { "type": "text", "text": "这张图片里有什么？" },
      { "type": "image_url", "image_url": { "url": "..." } }
    ]
  }]
}
```

---

## 2. 模型选型决策树

```
├── 简单任务 → GPT-3.5 / 轻量模型（便宜、快）
├── 复杂任务 → GPT-4 / Claude 3.5（贵、准）
├── 需要私有化 → 开源模型（LLaMA / Qwen / DeepSeek）
├── 中文为主 → 国产模型优先
└── 预算有限 → 小模型 + Prompt 优化
```

### 四维评估

| 维度 | GPT-3.5 | GPT-4 |
|------|---------|-------|
| 效果 | 简单任务够用 | 复杂任务优秀 |
| 成本 | ~$0.5/百万token | ~$30/百万token（60倍！）|
| 延迟 | 快 | 慢 3-5 倍 |
| 安全 | 云端 | 云端（敏感场景需私有化）|

---

## 3. 分级路由策略（真实项目经验）

```
用户问题 → 意图分类器（GPT-3.5）
  ├── 简单 → GPT-3.5 回答
  ├── 复杂 → GPT-4 回答
  └── 敏感 → 转人工
```

**效果**：准确率降 2%，成本降 70%

---

## 4. 关键经验

- **能用小模型解决的，绝不用大模型**
- **先用 Prompt 优化压榨小模型，不够再上大模型**
- **法律/金融/医疗 → 必须考虑数据合规和私有化部署**
- **数据脱敏**：敏感信息先替换再送模型
